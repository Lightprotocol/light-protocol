# Plan: xtask fetch-keypair-txs

**Date:** 2026-03-19

## IMPORTANT

- Add a new xtask subcommand `fetch-keypair-txs`
- For each provided public key, query `getSignaturesForAddress` and count txs in the last N minutes

---

## Context

The user wants to monitor how many transactions a set of addresses sent on Solana over a series of time buckets. The output should show tx counts per X-minute interval (not just a total), giving a temporal view of activity across keypairs.

## Implementation

### `fetch_keypair_txs.rs`

Pattern mirrors `fetch_failed_txs.rs`.

**Options:**
```rust
pub struct Options {
    /// One or more base58 public keys (positional, space-separated)
    #[clap(required = true)]
    pubkeys: Vec<String>,

    /// Bucket size in minutes (default: 10)
    #[clap(long, default_value_t = 10)]
    minutes: u64,

    /// Number of buckets to show going back in time (default: 6)
    #[clap(long, default_value_t = 6)]
    buckets: u64,

    /// Network: mainnet | devnet | testnet | local
    #[clap(long, default_value = "mainnet")]
    network: String,

    /// Custom RPC URL (overrides --network)
    #[clap(long)]
    rpc_url: Option<String>,
}
```

**Core logic:**
1. Compute `total_lookback = minutes * buckets` seconds
2. For each pubkey: call `client.get_signatures_for_address(&pubkey)` вЂ” returns up to 1000 most-recent confirmed sigs
3. Build `buckets` time slots: `[now - k*minutes*60 .. now - (k-1)*minutes*60)` for k = 1..=buckets (slot 0 = most recent)
4. Bin each sig's `block_time` into the appropriate slot; count per slot
5. Drop any sig older than `now - total_lookback`

**Output:** one row per pubkey, alternating count + TPS columns per bucket, plus a Total column at the end.

- Each bucket pair: count column (`-10m`) + TPS column (`TPS`) where `TPS = count / (minutes * 60)`
- **Total**: sum of txs across all buckets (no total TPS column)

**Addresses** are shortened (`first6...last3`) to keep the table readable.

**Column generation (dynamic):** header row = `["Address", "-10m", "TPS", "-20m", "TPS", ..., "Total"]`, built programmatically in a loop. Use `tabled::builder::Builder` since column count depends on `--buckets`.

**Reuse from existing code:**
- `network_to_url()` вЂ” copy from `fetch_failed_txs.rs`
- `CommitmentConfig::confirmed()`
- `chrono::Utc`, `tabled`

**Note:** `tabled` requires a statically-known column set. Since bucket count is dynamic, build the table with `tabled::builder::Builder` (row-by-row string building) instead of a `#[derive(Tabled)]` struct.
