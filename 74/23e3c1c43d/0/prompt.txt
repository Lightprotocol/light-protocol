# Plan: Add network fee on V1 output appends

## Context

V1 state tree outputs (appends) do not charge a network fee. V2 outputs charge 5,000 lamports (once per tree) via `set_network_fee_v2()`. This is inconsistent -- V1 outputs should also charge the network fee. Since V1 is deprecated, this is a minor fee increase for remaining V1 users, encouraging migration.

## Changes

### 1. Add `set_network_fee_v1` call in V1 output path

**File:** `programs/system/src/processor/create_outputs_cpi_data.rs`

In the `AcpAccount::StateTree` branch (line ~93-109), after extracting `merkle_context`, add:

```rust
context.set_network_fee_v1(merkle_context.network_fee, current_index as u8)?;
```

This uses the existing additive `set_network_fee_v1` method, consistent with V1 input fee charging. The fee goes to the state tree account (same destination as the rollover fee for that tree).

Note: `set_network_fee_v1` is additive per call. For outputs, it's called once per unique output tree (since the branch only executes when `current_index` changes). So a tx with outputs to 1 V1 tree = +5,000; outputs to 2 V1 trees = +10,000.

### 2. Update `assert_transaction_params` fee calculation

**File:** `sdk-libs/client/src/fee.rs`

Add a new field to `TransactionParams`:
```rust
pub v1_output_compressed_accounts: u8,
```

Update the network fee calculation (~line 73) to also account for V1 output trees. Keep the existing `else if` between V1 and V2 input checks:
```rust
// Input network fee (V1 per-input-tree, V2 once-per-tree, mutually exclusive)
if transaction_params.v1_input_compressed_accounts != 0 {
    network_fee += transaction_params.fee_config.network_fee as i64
        * transaction_params.v1_input_compressed_accounts as i64;
} else if transaction_params.v2_input_compressed_accounts {
    network_fee += transaction_params.fee_config.network_fee as i64;
}
// Output network fee (V1 only, per-output-tree)
if transaction_params.v1_output_compressed_accounts != 0 {
    network_fee += transaction_params.fee_config.network_fee as i64
        * transaction_params.v1_output_compressed_accounts as i64;
}
```

Note: V1 output fee is per-tree (not per-output-account), so this field should represent the number of unique V1 output trees.
